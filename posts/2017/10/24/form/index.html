form

<form>标签用于创建供用户输入的 HTML 表单。

常用属性

属性 值 描述
accept-charset character_set 规定服务器可处理的表单数据字符集。
action URL 规定当提交表单时向何处发送表单数据。
enctype application，x-www-form-urlencoded，multipart，form-datatext，plain 规定在向服务器发送表单数据之前如何对其进行编码。（适用于 method=”post” 的情况）
method get，post 规定用于发送表单数据的 HTTP 方法。
name text 规定表单的名称。
novalidate novalidate 如果使用该属性，则提交表单时不进行验证。
target _blank，_self，_parent，_top 规定在何处打开 action URL。

GET 和 POST 的区别

  • GET：从指定的资源请求数据。
  • POST：向指定的资源提交要被处理的数据。
GET POST
无害 后退按钮/刷新 数据会被重新提交（浏览器应该告知用户数据会被重新提交）。
可收藏为书签 书签 不可收藏为书签
能被缓存 缓存 不能缓存
application/x-www-form-urlencoded 编码类型 application/x-www-form-urlencoded 或 multipart/form-data。为二进制数据使用多重编码。
参数保留在浏览器历史中。 历史 参数不会保存在浏览器历史中。
当GET 方法发送数据时要向 URL 添加数据，URL 的长度是受限制的，最大长度是 2048 个字符。 数据长度限制 无限制。
只允许 ASCII 字符。 数据类型限制 没有限制。也允许二进制数据。
GET 的安全性相对较差，因为所发送的数据是 URL 的一部分，发送密码或其他敏感信息时不要使用 GET。 安全性 POST 比 GET 更安全，因为参数不会被保存在浏览器历史或 Web 服务器日志中。
数据在 URL 中对所有人都是可见的。 可见性 数据不会显示在 URL 中。

属性 值 描述
accept mime_type 规定通过文件上传来提交的文件的类型。
type button / checkbox / color / date / datetime / datetime-local / email / file / hidden / image / month / number / password / radio / range / reset / search / submit / tel / text / time / url / week 规定 input 元素的类型。
name field name 定义 input 元素的名称。
value value 规定 input 元素的值。

隐藏字段

隐藏字段对于用户是不可见的。隐藏字段通常会存储一个默认值，可以用 JavaScript 修改。

  1. 在表单中插入隐藏域的目的在于收集或发送信息，以利于被处理表单的程序所使用。浏览者单击发送按钮发送表单的时候，隐藏域的信息也被一起发送到服务器。
  2. 有时候需要给用户传递信息，让他在提交表单时提交上来以确定用户身份，如 sessionkey。当然这些东西也能用 cookie 实现，但使用隐藏域就简单的多了．而且不会有浏览器不支持，用户禁用 cookie 的烦恼。
  3. 有时候一个 form 里有多个提交按钮，可以使用隐藏域分辨到底是按那一个按钮提交的。
  4. 有时候一个网页中有多个 form，多个 form 是不能同时提交的，但有时这些 form 确实相互作用，我们就可以在 form 中添加隐藏域来使它们联系起来。
